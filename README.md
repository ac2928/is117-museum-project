# Museum Project — The Evolution of Web Design

## Live Site
https://ac2928.github.io/is117-museum-project/


## Project Overview
This project is a digital museum experience that explores the evolution of web design from the 1990s to the present. The goal is not just to present information, but to guide the visitor through a structured narrative showing how web design developed from technical limitation into a user-centered discipline.


## Core Narrative
The central argument of this museum is that web design evolved from solving technical constraints to shaping human experience.

The visitor moves through a guided timeline:
- Early Web (1990s): functionality over design
- Web 2.0 (2000s): interaction and engagement
- Modern Web: user experience and responsiveness
- Future: AI-driven and personalized design

The final section, “Why It Matters,” connects this evolution to real-world impact and user behavior.


## Design Framework

### Design Style — Swiss Minimal
The site uses a Swiss design approach to ensure clarity, structure, and consistency.

This is reflected in:
- Clean grid-based layout
- Consistent spacing and alignment
- Minimal color palette (black, white, gray)
- Clear typographic hierarchy

The goal was to remove visual noise and let the content guide the experience.


### Brand Archetype — Sage
The site follows the Sage archetype, focusing on education, clarity, and insight.

This influences:
- Informational tone
- Clear explanations
- Structured content delivery

The user experience is designed to feel like a curated educational exhibit.


### Cialdini Principle — Authority
Authority is used to increase credibility and engagement.

This is shown through:
- Structured explanations
- External references and links
- A museum-style presentation of information

This helps the user trust the content and continue exploring.


## AI Orchestration Process

This project was built using a structured AI workflow rather than a single prompt.

### Phase 1 — Planning
- Defined the topic and narrative before building
- Identified key sections and flow of the museum
- Studied reference design systems and example museum sites

### Phase 2 — Development
- Used AI to generate layout structure and section organization
- Built the site using HTML and CSS
- Implemented a consistent design system

### Phase 3 — Iteration and Refinement
- Fixed image loading issues by switching to reliable sources
- Adjusted layout spacing to avoid oversized or narrow sections
- Improved visual balance and readability
- Implemented a scroll-based navigation bar that changes color based on background

### Phase 4 — Evaluation
- Reviewed the site for clarity, consistency, and usability
- Ensured the design aligned with Swiss principles
- Refined content to improve narrative flow

AI was used as a tool for refinement and problem-solving, but all final decisions were made intentionally.


## Reflection

This project demonstrated the importance of planning, structure, and iteration in design.

Instead of building randomly, I followed a spec-driven approach where each decision was made with a clear goal. Breaking the project into stages made it easier to identify problems and improve the site step by step.

One key takeaway was that AI is most effective when used in a controlled workflow. Rather than relying on a single output, I used AI to refine layout, fix issues, and improve design quality over time.

Overall, this project shows how intentional design and structured development can produce a clear, engaging, and effective digital experience.


## AI Curator Review Summary

The site was reviewed using AI acting as a museum curator. The feedback highlighted strengths in structure and clarity, while also identifying areas for improvement such as narrative flow and engagement.

Based on this feedback, the site was refined to improve spacing, readability, and overall coherence, resulting in a more polished and purposeful museum experience.
